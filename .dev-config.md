## 📝 Convenções de Código

### Backend (PHP/Laravel)

#### Nomenclatura
- Models: Singular, PascalCase (e.g., `Member`, `Invoice`)
- Controllers: PascalCase + Controller (e.g., `MemberController`)
- Methods: camelCase (e.g., `store`, `updateStatus`)
- Variables: camelCase (e.g., `$userId`, `$memberData`)

#### Padrões
```php
// ✅ BOM: Controller com Form Request
public function store(StoreMemberRequest $request)
{
    $member = Member::create($request->validated());
    return new MemberResource($member);
}

// ✅ BOM: Service para lógica complexa
class InvoiceService
{
    public function generateMonthlyInvoices(): void
    {
        // Lógica de negócio aqui
    }
}

// ❌ EVITAR: Lógica no controller
public function store(Request $request)
{
    // 50 linhas de código...
}
```

#### Relationships
```php
// Model Member
public function user(): BelongsTo
{
    return $this->belongsTo(User::class);
}

public function invoices(): HasMany
{
    return $this->hasMany(Invoice::class);
}
```

### Frontend (TypeScript/React)

#### Nomenclatura
- Components: PascalCase (e.g., `MemberForm`, `StatusBadge`)
- Files: match component name (e.g., `MemberForm.tsx`)
- Hooks: camelCase com 'use' prefix (e.g., `useMembers`, `useAuth`)
- Types/Interfaces: PascalCase (e.g., `Member`, `ApiResponse`)

#### Padrões
```typescript
// ✅ BOM: Functional component com TypeScript
interface MemberFormProps {
  member?: Member;
  onSave: (data: MemberFormData) => void;
  onCancel: () => void;
}

export const MemberForm: React.FC<MemberFormProps> = ({ 
  member, 
  onSave, 
  onCancel 
}) => {
  // Component code
};

// ✅ BOM: Custom hook para API
export const useMembers = () => {
  const [members, setMembers] = useState<Member[]>([]);
  const [loading, setLoading] = useState(false);
  
  const fetchMembers = async () => {
    setLoading(true);
    try {
      const response = await api.get('/members');
      setMembers(response.data);
    } catch (error) {
      // Handle error
    } finally {
      setLoading(false);
    }
  };
  
  return { members, loading, fetchMembers };
};
```

#### Estado (State Management)
- Local state: `useState` para UI simples
- Server state: React Query para dados da API
- Global state: Context API ou Zustand se necessário

## 🔐 Segurança

### Backend
- ✅ Sempre usar Form Requests para validação
- ✅ Policies para autorização
- ✅ Sanctum tokens para autenticação
- ✅ Rate limiting em rotas públicas
- ✅ Sanitize user input
- ✅ HTTPS em produção

### Frontend
- ✅ Nunca expor tokens em URLs
- ✅ XSS protection (escape user content)
- ✅ Validar no client E no server
- ✅ CSRF token em forms

## 📊 Base de Dados

### Convenções
- Tabelas: plural, snake_case (e.g., `members`, `invoice_items`)
- Colunas: snake_case (e.g., `first_name`, `due_date`)
- Foreign keys: `{model}_id` (e.g., `user_id`, `member_id`)
- Timestamps: sempre incluir `created_at`, `updated_at`

### Migrations
```php
// ✅ BOM: Migration bem estruturada
Schema::create('members', function (Blueprint $table) {
    $table->id();
    $table->foreignId('user_id')->constrained()->onDelete('cascade');
    $table->string('number')->unique();
    $table->date('birthdate');
    $table->enum('status', ['active', 'inactive', 'suspended'])->default('active');
    $table->timestamps();
    $table->softDeletes();
});
```

## 🚀 Workflow de Desenvolvimento

### 1. Nova Feature
```bash
# 1. Criar branch
git checkout -b feature/nome-da-feature

# 2. Implementar (backend primeiro, depois frontend)

# 3. Testar localmente

# 4. Commit (doc atualiza automaticamente)
git add .
git commit -m "feat: descrição da feature"

# 5. Push e criar PR
git push origin feature/nome-da-feature
```

### 2. Commits Semânticos
```
feat: nova funcionalidade
fix: correção de bug
docs: alteração na documentação
style: formatação, ponto e vírgula
refactor: refatoração de código
test: adição de testes
chore: alteração em build, configs
```

### 3. Pull Request Checklist
- [ ] Código segue convenções
- [ ] Testes passam
- [ ] Documentação atualizada
- [ ] Design system seguido (se frontend)
- [ ] API documentada (se backend)
- [ ] Reviewed por pelo menos 1 pessoa
